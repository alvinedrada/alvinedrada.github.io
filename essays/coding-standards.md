## Standards in English and Coding

maybe kevin right. wat if me just type rest of essay like dis? u understand still? but it ok cuz save time. Just kidding, please don't talk like Kevin. English, like many other languages, has its own grammatical structure and standards that people follow. We learn often these grammar rules in whatever language you learn as children in grade school in order to help us understand others and for others to understand us. These grammar rules helps us to learn the language and how to speak it better. Imagine throwing all of that away just to "save some time". Programming languages also have coding standards that coders should be following. There are tons of tools that we could use to improve the quality of our code, for example Checkstyle and ESLint. These tools can do so much to help us as programmers, such as reducing errors and improving readability. Coding without any coding standard tools would be similar to speaking like Kevin, technically it works and could save you some time, but using the conventional grammar rules that everyone else follows helps others understand you better. Just as grammar helps us learn a linguistic language, coding standards help you learn a programming language.

## Coding Standard Tools

ICS 211 introduced me to my first coding standard. Checkstyle for Java. Checkstyle and I did not have the greatest relationship. Everytime I did one of my assignments for that class, it felt like a nuisance to check and figure out every single thing that was wrong with my code. There were some assignments where I would just submit it and ignore the warnings from Checkstyle. Although it did teach me some things about formatting that made the code easier to read, I was happy to be done with that after the semester ended.

I probably should have realized that coding standard tools would have followed me throughout college and will probably follow me throughout my career. It has been about a week since I first started using ESLint with IntelliJ. First impressions: there were a lot of steps to install and to get this thing working correctly. At first I was having trouble with getting the "one time" tasks to work for the Javascript coding standards, my settings were not saving them and I would have to change them every time I made a new project. It turns out I had to change the setting profile to the IDE default instead of it being the project default. I also find it tedious as in order for the ESLint to work, you have to download some files into your project directory each time you make a new project. I might have to find an efficient way to do this in the future.
