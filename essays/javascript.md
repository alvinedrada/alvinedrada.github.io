## Javascript

People usually say that learning a programming language is hard. JavaScript is an exception. This might be due to my prior experience with Java, which is extremely similar to JavaScript. I remember my first ICS professor saying something along the lines of "The first programming language is always the hardest to learn, then the next ones will be easier." This rang true for me, as JavaScript was easy for me to pick up. If you already know Java, you need to learn JavaScript. While learning JavaScript, there will be familiar elements as the ones you see in Java, but it will be so much more easier and fluid to code. Even if you do not know any other programming languages, I think that JavaScript can also be a good start because to me it seemed really beginner-friendly. I was able to complete freeCodeCamp.org's Basic JavaScript section over the course of two days! There was also new things I learned with ECMAScript 6 (ES6), which is a standardized version of JavaScript. Overall, I think that JavaScript should be at the top of your list of languages to learn as it can teach you the fundamentals of programming and also teach you new things you did not learn with your other languages. 

## Athletic software engineering

This semester I am learning of the term "Athletic Software Engineering". The basic premise of it is it pretty much puts you in a pressured situation where you have to solve a problem within an allotted time frame. It is similar to what athletes would do, where as an athlete you would exercise to get faster and stronger, an engineer would do this to get better at software engineering skills. To do this, we do WODs or "Workout of the Day" which requires us to solve a problem correctly and in the time limit. It is an all or nothing system so if you do not solve the problem in time, you will lose all the points. Today, I had my first real WOD. I barely passed, as I came within 3 minutes of the time limit. My experience with it was pretty stressful, but in the end I felt good that I actually completed the problem on time. We also had practice WODs that I did beforehand and I think that those practice times really helped with improving my skills and confidence to complete the problem in time, even if it took a couple tries. This style of learning is completely new to me, but I look forward to trying it more this semester. I am excited to see how much better I become at WODs and athletic software engineering at the end of this course.
