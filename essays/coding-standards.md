## Standards in English and Coding

maybe kevin right. wat if me just type rest of essay like dis? u understand still? but it ok cuz save time. Just kidding, please don't talk like Kevin. English, like many other languages, has its own grammatical structure and standards that people follow. We learn often these grammar rules in whatever language you learn as children in grade school in order to help us understand others and for others to understand us. These grammar rules helps us to learn the language and how to speak it better. Imagine throwing all of that away just to "save some time". Programming languages also have coding standards that coders should be following. There are tons of tools that we could use to improve the quality of our code, for example Checkstyle and ESLint. These tools can do so much to help us as programmers, such as reducing errors and improving readability. Coding without any coding standard tools would be similar to speaking like Kevin, technically it works and could save you some time, but using the conventional grammar rules that everyone else follows helps others understand you better. Just as grammar helps us learn a linguistic language, coding standards help you learn a programming language.
